description: Register a roadmap node as in-progress (on integration branch, then feature branch)

Thin stub — adds a registry entry before any implementation starts. Generated by specyrd {{SPECYRD_VERSION}}.

Replace `CODENAME` and `NODE_ID` with the values from the roadmap node.

From the **repository root**:

```bash
cd "$(git rev-parse --show-toplevel)"
```

1. Check out your **integration branch** (e.g. `main` or `dev`), pull, ensure a **clean** tree.

2. Add an entry to `roadmap/registry.yaml`:

```yaml
entries:
  - codename: CODENAME
    node_id: NODE_ID
    branch: feature/rm-CODENAME
    touch_zones:
      - path/to/affected/dir/
    started: "YYYY-MM-DD"
    owner: "your name or agent ID"
```

3. Validate, commit **on the integration branch**, push so PMs see the claim:

```bash
specy-road validate
git add roadmap/registry.yaml
git commit -m "chore(rm-CODENAME): register as in-progress"
git push origin <integration-branch>
```

4. Create the implementation branch:

```bash
git checkout -b feature/rm-CODENAME
```

No implementation before the registration commit on the integration branch. See `docs/dev-workflow.md` and `docs/git-workflow.md` for full protocol.
