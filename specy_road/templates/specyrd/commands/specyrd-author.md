Thin stub — roadmap YAML is the canonical source of truth. Generated by specyrd {{SPECYRD_VERSION}}.

From the **repository root**:

```bash
cd "$(git rev-parse --show-toplevel)"
```

1. Find the target chunk file under `roadmap/phases/` (or create a new one and add it to `roadmap/roadmap.yaml` under `includes`). Use `specy-road list-nodes` to see which chunk holds each id.

2. Prefer **`specy-road add-node`** (see `specy-road add-node -h`) for append-only tasks, or edit YAML by hand following `docs/roadmap-authoring.md`. Minimum required fields when authoring by hand:

```yaml
- id: Mx.y          # immutable; never renumber; gaps allowed
  parent_id: Mx
  type: milestone   # vision | phase | milestone | task
  title: "Human-readable label"
  codename: kebab-slug   # unique; used for branch naming
  execution_milestone: Human-led   # Human-led | Agentic-led | Mixed
  status: Not Started
  touch_zones:
    - path/to/affected/dir/
```

3. For `agentic` sub-tasks, include the five-field `agentic_checklist` (see `docs/roadmap-authoring.md — Writing implementable agentic tasks`).

4. Validate and regenerate the markdown index:

```bash
specy-road validate
specy-road export
```

5. Commit: `chore(roadmap): add <node-id> <short title>`

See `docs/pm-workflow.md` for the full PM authoring workflow.
